# 国際化（i18n）ガイド

NetTools Hub は世界中のユーザーを対象としています。本ドキュメントでは翻訳の仕組みと、新しい翻訳の追加・改善方法を説明します。

> 🇯🇵 日本語 · 🇬🇧 [English](./I18N.md) · 🇨🇳 [简体中文](./I18N.zh.md)

---

## 1. コードベースにおける i18n の仕組み

### 1.1 1 ファイル 3 言語

すべての UI 文字列は **1 つのファイル** [`src/lib/i18n.ts`](../src/lib/i18n.ts) に集約されています。3 列のテーブルです：

```ts
const translations = {
  en: { "hero.title1": "All the best", "hero.title2": "open-source network tools", … },
  zh: { "hero.title1": "最棒的开源", "hero.title2": "网络工具一站直达", … },
  ja: { "hero.title1": "ベストな",       "hero.title2": "オープンソースネットワークツール", … },
};
```

現在の言語はランタイムで `?lang=` URL パラメータから読み取られます。**Cookie も localStorage も使いません** —— 共有可能でステートレスなリンクを保つためです。

### 1.2 `t()` 関数

```ts
import { t } from "@/lib/i18n";

t("en", "hero.title1");                       // "All the best"
t("zh", "hero.title1");                       // "最棒的开源"
t("ja", "hero.title1");                       // "ベストな"

t("en", "category.count", { count: 12 });     // "12 projects"
t("zh", "category.count", { count: 12 });     // "12 个项目"
t("ja", "category.count", { count: 12 });     // "12 プロジェクト"
```

プレースホルダの構文は **`{name}`**（中括弧の前後にスペースなし）。

### 1.3 欠落キー

ある言語でキーが欠けている場合、`t()` は英語にフォールバックし、コンソールに警告を出力します。サイトをレンダリング可能に保ちつつ、未翻訳箇所を通知します。

## 2. 新しい言語を追加する

> スペイン語 / フランス語 / 韓国語 / ロシア語 / アラビア語 / … を追加したい方は大歓迎です！手順は次のとおりです。

### ステップ 1 — i18n ファイルを開く

```bash
$EDITOR src/lib/i18n.ts
```

### ステップ 2 — 言語を登録する

[`Lang`](../src/lib/i18n.ts) 共用体型にコードを追加：

```ts
export type Lang = "en" | "zh" | "ja" | "es";   // ← "es" を追加
```

同じファイル内の `LANG_OPTIONS`（言語スイッチャーで使う配列）にも追加：

```ts
export const LANG_OPTIONS = [
  { value: "en", label: "EN" },
  { value: "zh", label: "中文" },
  { value: "ja", label: "日本語" },
  { value: "es", label: "Español" },   // ← 追加
] as const;
```

### ステップ 3 — 翻訳テーブルに新しい列を追加する

**`en` と同じキー** を持つ新しいエントリを追加：

```ts
const translations = {
  en: { … },
  zh: { … },
  ja: { … },
  es: {                                          // ← 新規
    "site.title": "NetTools Hub",
    "hero.title1": "Todas las mejores",
    "hero.title2": "herramientas de red de código abierto",
    // …
  },
};
```

> 💡 最速の方法は `en` ブロック全体をコピーして 1 つずつ翻訳すること。欠落キーは英語にフォールバックするため、下書き PR を投げて徐々に埋めていっても OK です。

### ステップ 4 — `isValidLang()` と `getLangFromUrl()` を更新する

`src/lib/i18n.ts` の 2 つのヘルパーを更新：

```ts
export function isValidLang(v: string | null | undefined): v is Lang {
  return v === "en" || v === "zh" || v === "ja" || v === "es";   // ← 更新
}
```

### ステップ 5 — 対応言語の README を追加する

```bash
cp README.md README.es.md
# 本文を翻訳
```

GitHub は `README.<lang>.md` を自動検出します。

### ステップ 6 — 言語スイッチャーにエントリを追加する

ステップ 2 で完了済みです。

### ステップ 7 — PR を作成する

PR 説明に以下を記載：

- 追加した言語
- ネイティブ / 堪能であることの宣言
- README も翻訳したか（推奨）

## 3. 既存翻訳を改善する

誤字や不自然な言い回しを発見した場合、2 つの選択肢があります：

### オプション A — i18n ファイルを直接編集

```bash
$EDITOR src/lib/i18n.ts
```

対応するキー（例：`"empty.description"`）を見つけて値を編集、コミット、プッシュ、PR 作成。

### オプション B — 先に Issue を立てる

言い回しが不安な場合は、[翻訳 Issue](../../issues/new?template=translation.yml) を開いて議論。

---

## 4. 翻訳スタイルガイド

| ガイドライン | 例 |
|---|---|
| **短く** —— UI ラベルは幅制約あり | "Explore" / "探索" / "探索"（3 文字） |
| 必要がなければ **技術用語を翻訳しない** | `Clash`、`sing-box`、`WebSocket`、`CDN` はそのまま |
| **トーンを合わせる** —— 簡潔・プロ・友好的 | 元になければ絵文字を追加しない |
| **プレースホルダを保持** —— `{count}`、`{name}` は必須 | `t(lang, "category.count", { count })` |
| **LANG_OPTIONS のラベルはその言語で** | "日本語" であり "Japanese" ではない |

---

## 5. 翻訳を検証する

```bash
pnpm dev     # http://localhost:8080/?lang=es
```

1. 各ページ（`/`、`/explore`）を開く
2. 右上の Globe アイコンをクリック
3. 新しい言語を選択
4. すべてのラベル、エラー状態、404 ページ、空状態、"Clear filters" ボタンなどをチェック
5. 文字列が欠落していると英語にフォールバックしコンソールに警告が出るので修正
